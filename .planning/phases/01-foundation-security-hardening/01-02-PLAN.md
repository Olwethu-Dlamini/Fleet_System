       ```javascript
       const MAX_RETRIES = 3;

           await connection.beginTransaction();
       ```

    Replace the `generateJobNumber` static method in Job.js. The current implementation uses `SELECT MAX(job_number) ... + 1` which has a race condition when two jobs are created simultaneously — both read the same MAX, both increment to the same number, and one write fails with a unique key error (or worse, if there is no unique key, duplicates are silently created).

    **New implementation using atomic sequence table:**
    ```javascript
    static async generateJobNumber() {
      const year = new Date().getFullYear();

      // Step 1: Atomic increment — LAST_INSERT_ID trick returns the NEW value
      // This is a single atomic database operation; no race condition is possible
      await db.query(
        `UPDATE job_number_sequences
         SET counter = LAST_INSERT_ID(counter + 1)
         WHERE year = ?`,
        [year]
      );

      // Step 2: Retrieve the value we just set (same connection context)
      const [[seq]] = await db.query(`SELECT LAST_INSERT_ID() AS counter`);

      // Format: JOB-2026-0001
      return `JOB-${year}-${String(seq.counter).padStart(4, '0')}`;
    }
    ```

    The `LAST_INSERT_ID(expr)` trick: when passed an expression, MySQL/MariaDB stores that expression's result as the last insert ID for the current connection, and the subsequent `SELECT LAST_INSERT_ID()` retrieves it. This is guaranteed atomic — no two connections can get the same value.

    **Important:** The `job_number_sequences` row for the current year is created by the Plan 01 migration (seeded from existing data). If for any reason the row is missing (e.g., January 1st rollover to a new year), add an INSERT fallback before the UPDATE:
    ```javascript
    // Ensure row exists for this year (handles year rollover)
    await db.query(
      `INSERT IGNORE INTO job_number_sequences (year, counter) VALUES (?, 0)`,
      [year]
    );
    ```

    Full method:
    ```javascript
    static async generateJobNumber() {
      const year = new Date().getFullYear();
      // Ensure row exists for this year (handles year rollover, safe if row already exists)
      await db.query(
        `INSERT IGNORE INTO job_number_sequences (year, counter) VALUES (?, 0)`,
        [year]
      );
      // Atomic increment: no two calls can return the same counter value
      await db.query(
        `UPDATE job_number_sequences SET counter = LAST_INSERT_ID(counter + 1) WHERE year = ?`,
        [year]
      );
      const [[seq]] = await db.query(`SELECT LAST_INSERT_ID() AS counter`);
      return `JOB-${year}-${String(seq.counter).padStart(4, '0')}`;
    }
    ```
