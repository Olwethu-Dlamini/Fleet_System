**Step 1: DB migration in server.js startup**

Add a self-executing migration block inside the `if (require.main === module)` guard in server.js (after the DB connection check, before starting cron). Use `CREATE TABLE IF NOT EXISTS` for idempotent execution:

```sql
CREATE TABLE IF NOT EXISTS notifications (
  id              INT AUTO_INCREMENT PRIMARY KEY,
  tenant_id       INT NOT NULL,
  user_id         INT NOT NULL,
  job_id          INT,
  type            VARCHAR(50) NOT NULL,
  title           VARCHAR(255) NOT NULL,
  body            TEXT NOT NULL,
  is_read         BOOLEAN NOT NULL DEFAULT FALSE,
  created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  INDEX idx_notifications_tenant_user (tenant_id, user_id),
  INDEX idx_notifications_job (job_id),
  INDEX idx_notifications_created (created_at)
);

CREATE TABLE IF NOT EXISTS notification_preferences (
  id              INT AUTO_INCREMENT PRIMARY KEY,
  tenant_id       INT NOT NULL,
  user_id         INT NOT NULL UNIQUE,
  email_enabled   BOOLEAN NOT NULL DEFAULT TRUE,
  push_enabled    BOOLEAN NOT NULL DEFAULT TRUE,
  created_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_notif_prefs_user (user_id)
);
```

Run each CREATE TABLE as a separate `db.query()` call. Log success with `logger.info('Notification tables ensured')`.

**sendTopicNotification(topic, title, body, data = {})**:
- Lazy-load firebase: `let admin; try { admin = require('../config/firebase'); } catch(e) { logger.warn('Firebase not configured — skipping FCM'); return; }`
- Build message: `{ notification: { title, body }, data, topic }`

- Catch errors: `logger.warn({ topic, err: err.message }, 'FCM send failed — skipping');` (per CONTEXT: no retry)

**checkUpcomingJobs()** (NOTIF-02):
- Query jobs starting in 10-20 minute window (gives 15-min lead time accounting for cron interval)
- Join `jobs` -> `job_assignments` (or `job_technicians`) -> `users` for recipient info
- LEFT JOIN `notification_preferences` for email_enabled check
- Dedup: `NOT EXISTS (SELECT 1 FROM notifications WHERE job_id=? AND user_id=? AND type='job_starting_soon' AND created_at > DATE_SUB(NOW(), INTERVAL 20 MINUTE))`
- For each matched job/user:
  1. INSERT into `notifications` table (in-app record)
  2. Call `sendTopicNotification('driver_{userId}', 'Job Starting Soon', '{job_title} starts in ~15 minutes', { jobId: String(job.id), type: 'job_starting_soon' })`
  3. If `email_enabled`: call `EmailService.sendJobNotification(...)` with user email
- Also notify schedulers: query users with role in ('admin','scheduler','dispatcher') for same tenant, send to `scheduler_{userId}` topic

**checkOverdueJobs()** (NOTIF-03):
- Query jobs where `CONCAT(scheduled_date, ' ', scheduled_time_end) < DATE_SUB(NOW(), INTERVAL 5 MINUTE)` AND `current_status NOT IN ('completed','cancelled')`
- Same dedup pattern with type='job_overdue'
- Notify assigned driver AND all schedulers for the tenant
- Driver message: "Your job {job_title} is overdue"
- Scheduler message: "Job {job_number} assigned to {driver_name} is overdue"

**cleanOldNotifications()** (30-day retention per CONTEXT):
- `DELETE FROM notifications WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY)`
- Called from a daily cron schedule

Export: `{ sendTopicNotification, checkUpcomingJobs, checkOverdueJobs, cleanOldNotifications }`
