1. Append to `vehicle_scheduling.sql` two CREATE TABLE IF NOT EXISTS statements:
   - `time_extension_requests` table: id (auto_increment unsigned), tenant_id (unsigned NOT NULL), job_id (unsigned NOT NULL), requested_by (unsigned NOT NULL), duration_minutes (unsigned NOT NULL), reason (text NOT NULL), status (enum pending/approved/denied DEFAULT pending), denial_reason (text DEFAULT NULL), approved_denied_by (unsigned DEFAULT NULL), approved_denied_at (timestamp NULL), selected_suggestion_id (unsigned DEFAULT NULL), created_at (timestamp DEFAULT current_timestamp), updated_at (timestamp DEFAULT current_timestamp ON UPDATE current_timestamp). Indexes: idx_ter_job_status(job_id, status), idx_ter_tenant(tenant_id), idx_ter_requested_by(requested_by).
   - `reschedule_options` table: id (auto_increment unsigned), request_id (unsigned NOT NULL), tenant_id (unsigned NOT NULL), type (enum push/swap/custom NOT NULL), label (varchar 100 NOT NULL), changes_json (text NOT NULL), created_at (timestamp DEFAULT current_timestamp). Indexes: idx_ro_request(request_id), idx_ro_tenant(tenant_id).

2. Create `timeExtensionService.js` with pino child logger (`service: 'timeExtensionService'`). Static methods:

   a. `createRequest({ jobId, requestedBy, durationMinutes, reason, tenantId })`:
      - Get connection, begin transaction
      - Verify job exists, is in_progress, belongs to tenant: `SELECT id, job_number, scheduled_date, scheduled_time_start, scheduled_time_end, estimated_duration_minutes FROM jobs WHERE id=? AND tenant_id=? AND current_status='in_progress' FOR UPDATE`
      - Verify no pending request: `SELECT id FROM time_extension_requests WHERE job_id=? AND status='pending' LIMIT 1 FOR UPDATE` — if exists, throw 409 error
      - Verify requestedBy is assigned: `SELECT 1 FROM job_technicians WHERE job_id=? AND user_id=? UNION SELECT 1 FROM job_assignments WHERE job_id=? AND driver_id=? LIMIT 1` — if empty, throw 403
      - INSERT into time_extension_requests
      - COMMIT transaction, release connection
      - Call `analyzeImpact()` (outside transaction, read-only)
      - Call `_buildSuggestions()` with impact results
      - INSERT reschedule_options rows for each suggestion
      - Call `_notifySchedulers()` with job info, requestId, tenantId
      - Return { request, affectedJobs, suggestions }

   b. `analyzeImpact(jobId, newEndTime, scheduledDate, tenantId)`:
      - Query: SELECT DISTINCT j.id, j.job_number, j.scheduled_date, j.scheduled_time_start, j.scheduled_time_end, j.estimated_duration_minutes, ja.vehicle_id, ja.driver_id FROM jobs j LEFT JOIN job_assignments ja ON ja.job_id=j.id WHERE j.tenant_id=? AND j.scheduled_date=? AND j.id!=? AND j.current_status NOT IN ('completed','cancelled') AND j.scheduled_time_start >= ? AND (ja.vehicle_id = (SELECT vehicle_id FROM job_assignments WHERE job_id=? LIMIT 1) OR EXISTS (SELECT 1 FROM job_technicians jt JOIN job_technicians jt2 ON jt2.user_id=jt.user_id AND jt2.job_id=j.id WHERE jt.job_id=?)) ORDER BY j.scheduled_time_start ASC
      - Return array of affected job objects

   c. `_buildSuggestions(sourceJob, extensionMinutes, affectedJobs, tenantId)`:
      - Always create "Push" suggestion: shift all affected jobs by extensionMinutes. Label: "Push all later jobs by {N} min". changes: array of {jobId, jobNumber, currentStart, currentEnd, newStart, newEnd}
      - Check for available swap drivers using query: SELECT DISTINCT u.id, u.first_name, u.last_name FROM users u WHERE u.tenant_id=? AND u.role IN ('driver','technician') AND u.is_active=1 AND u.id != ? (current driver) AND u.id NOT IN (SELECT jt.user_id FROM job_technicians jt JOIN jobs j2 ON j2.id=jt.job_id WHERE j2.scheduled_date=? AND j2.current_status NOT IN ('completed','cancelled') AND TIME_TO_SEC(j2.scheduled_time_start) < ? AND TIME_TO_SEC(j2.scheduled_time_end) > ?) — if found, create "Swap" suggestion with label "Reassign to {driverName}", changes: empty (no domino effect)
      - Always create "Custom" suggestion: label "Enter custom times", type custom, changes: []
      - Return suggestions array (2 or 3 items)
      - Use `addMinutesToTime(timeStr, minutes)` helper: split on ':', convert to total seconds, add minutes*60, format back to HH:MM:SS

   d. `_notifySchedulers(jobId, jobNumber, requestId, durationMinutes, tenantId)`:
      - Query schedulers/admins: SELECT id FROM users WHERE tenant_id=? AND role IN ('admin','scheduler','dispatcher') AND is_active=1
      - For each: INSERT INTO notifications (tenant_id, user_id, job_id, type, title, body) VALUES (?,?,?,'time_extension_requested','Time Extension Request','{jobNumber}: {durationMinutes} min extension requested')
      - For each: await NotificationService.sendTopicNotification(`scheduler_${scheduler.id}`, 'Time Extension Request', `${jobNumber}: ${durationMinutes} min extension requested`, { jobId: String(jobId), requestId: String(requestId), type: 'time_extension_requested' })

   e. `getActiveRequest(jobId, tenantId)`:
      - SELECT from time_extension_requests WHERE job_id=? AND tenant_id=? AND status='pending' LIMIT 1
      - If found, SELECT from reschedule_options WHERE request_id=? AND tenant_id=?
      - Return { request, suggestions } or { request: null }

   f. `approveRequest(requestId, suggestionId, customChanges, approvedBy, tenantId)`:
      - Get connection, begin transaction
      - SELECT request + original job FOR UPDATE
      - If suggestionId: SELECT changes_json FROM reschedule_options WHERE id=? AND request_id=?
      - Parse changes (from suggestion or customChanges)
      - UPDATE time_extension_requests SET status='approved', approved_denied_by=?, approved_denied_at=NOW(), selected_suggestion_id=?
      - UPDATE source job: SET scheduled_time_end = SEC_TO_TIME(TIME_TO_SEC(scheduled_time_end) + duration_minutes*60), estimated_duration_minutes = estimated_duration_minutes + duration_minutes
      - For each change in changes array: UPDATE jobs SET scheduled_time_start=?, scheduled_time_end=? WHERE id=? AND tenant_id=?
      - COMMIT, release connection
      - AFTER commit: call `_notifyAffectedParties(request, changes, tenantId)` — notify driver, all technicians on job, affected drivers

   g. `denyRequest(requestId, reason, deniedBy, tenantId)`:
      - UPDATE time_extension_requests SET status='denied', denial_reason=?, approved_denied_by=?, approved_denied_at=NOW() WHERE id=? AND tenant_id=? AND status='pending'
      - Notify driver + technicians on original job with type 'time_extension_denied'

   h. `_notifyAffectedParties(request, changes, tenantId)`:
      - Get driver_id + all technician user_ids for original job
      - Get driver_ids for all affected jobs (from changes)
      - Deduplicate user IDs
      - For each user: INSERT notification row, sendTopicNotification with type 'time_extension_approved'

1. Create `timeExtension.js` route file following existing route patterns (express.Router, verifyToken on all routes):

   a. POST `/` — Create time extension request
      - Auth: verifyToken (any authenticated user — service layer validates assignment)
      - Validation: body('job_id').isInt(), body('duration_minutes').isInt({min:1, max:480}), body('reason').isString().trim().isLength({min:10})
      - Handler: call TimeExtensionService.createRequest with req.body + req.user.id + req.user.tenant_id
      - Success: 201 with { success: true, request, affectedJobs, suggestions }
      - Error 409: already active request
      - Error 403: not assigned to job
      - Error 400: job not in_progress

   b. GET `/:jobId` — Get active request for a job
      - Auth: verifyToken
      - Validation: param('jobId').isInt()
      - Handler: call TimeExtensionService.getActiveRequest(jobId, tenantId)
      - Success: 200 with { success: true, request, suggestions } or { success: true, request: null }

   c. PATCH `/:id/approve` — Approve extension
      - Auth: verifyToken, requirePermission('jobs:update')
      - Validation: param('id').isInt(), body('suggestion_id').optional({nullable:true}).isInt(), body('custom_changes').optional({nullable:true}).isArray()
      - Handler: call TimeExtensionService.approveRequest(id, suggestion_id, custom_changes, req.user.id, req.user.tenant_id)
      - Success: 200 with { success: true, message: 'Extension approved' }

   d. PATCH `/:id/deny` — Deny extension
      - Auth: verifyToken, requirePermission('jobs:update')
      - Validation: param('id').isInt(), body('reason').optional().isString()
      - Handler: call TimeExtensionService.denyRequest(id, reason, req.user.id, req.user.tenant_id)
      - Success: 200 with { success: true, message: 'Extension denied' }

   All error handling: try/catch with logger.error and appropriate status codes (400, 403, 409, 500).

2. Update `index.js`: Add `const timeExtensionRoutes = require('./timeExtension');` to imports and `router.use('/time-extensions', timeExtensionRoutes);` to registrations, following the existing comment pattern.
