   - AppBar title: "Time Extension Request"

   Layout (SingleChildScrollView):

   a. Request Info Card (top):
      - Card showing: "Extension of {durationMinutes} min requested"
      - Reason text displayed below
      - Created at timestamp
      - Status chip (pending = orange, approved = green, denied = red)

   b. Impact Timeline Section (per TIME-03 visualization):
      - Section header: "Affected Jobs" with count badge
      - If no affected jobs: Text "No other jobs affected"
      - For each affected job from provider.affectedJobs:
        - ListTile showing job number, current start/end times
        - Small "arrow right" icon between current and proposed times (from push suggestion if available)
      - Style: use subtle dividers between items

   c. Suggestion Cards Section (per TIME-04, TIME-06):
      - Section header: "Rescheduling Options"
      - For each suggestion in provider.suggestions:
        - Card with: suggestion.label as title, suggestion.type as subtitle chip (push=blue, swap=green, custom=orange)
        - If type is push or swap: show changes list inside card (job number, newStart → newEnd)
        - If type is custom: show "You will enter custom times" text
        - Radio button or InkWell for selection
      - Track `int? _selectedSuggestionId` state variable
      - When "custom" is selected, show a section below with TextFormFields for each affected job: new start time (TimeOfDay picker or text input HH:MM), new end time. Store in `List<Map<String, dynamic>> _customChanges`.

   d. Action Buttons (bottom):
      - Row with two buttons:
        - "Deny" (OutlinedButton, red): shows dialog with optional reason TextFormField, then calls provider.denyRequest(requestId, reason). On success: SnackBar "Request denied", Navigator.pop.
        - "Approve" (ElevatedButton, primary color): disabled until a suggestion is selected. Calls provider.approveRequest(requestId, suggestionId: _selectedSuggestionId, customChanges: _customChanges if custom). On success: SnackBar "Extension approved — schedule updated", Navigator.pop.
      - Both buttons show loading indicator when provider.loading is true

   e. Loading state: show CircularProgressIndicator centered when provider.loading and request is null
   f. Empty state: show "No pending extension request for this job" when provider.activeRequest is null and not loading
   g. Error state: show provider.error in red Text widget with retry button
