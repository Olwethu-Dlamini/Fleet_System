    Under dependencies, add:

    Note: While the initial approach could use Socket.IO for real-time updates, use HTTP polling via `GpsService.getDriverLocations()` as the primary data source for v1 simplicity. The Socket.IO client dependency is added for future enhancement but the tracking screen will use a `Timer.periodic` polling approach calling the existing REST endpoint, which is simpler and more reliable for v1. The backend already handles tenant scoping and scheduler visibility gating on GET /drivers.

    **Create `lib/screens/gps/live_tracking_screen.dart`:**

    A StatefulWidget showing a GoogleMap with driver position markers, updated via polling.

    Structure:
    ```dart
    import 'dart:async';
    import 'package:flutter/material.dart';
    import 'package:google_maps_flutter/google_maps_flutter.dart';
    import 'package:vehicle_scheduling_app/config/theme.dart';
    import 'package:vehicle_scheduling_app/services/gps_service.dart';

    class LiveTrackingScreen extends StatefulWidget {
      const LiveTrackingScreen({super.key});
      @override
      State<LiveTrackingScreen> createState() => _LiveTrackingScreenState();
    }
    ```

    State fields:
    - `GoogleMapController? _mapController`
    - `final Map<MarkerId, Marker> _driverMarkers = {}`
    - `Timer? _pollTimer`
    - `bool _loading = true`
    - `String? _error`
    - `int _driverCount = 0`

    initState:
    - Call `_fetchDriverLocations()` immediately
    - Start `_pollTimer = Timer.periodic(Duration(seconds: 10), (_) => _fetchDriverLocations())`

    `_fetchDriverLocations()` method:
    - Call `GpsService.getDriverLocations()`
    - For each driver in the response list:
      - Create a `MarkerId('driver_${driver['driver_id']}')`
      - Parse `updated_at` (milliseconds since epoch) to DateTime
      - Calculate minutes ago: `DateTime.now().difference(updatedAt).inMinutes`
      - If minutes > 5: skip this driver (stale, don't show marker)
      - Create `Marker` with:
        - `position: LatLng(driver['lat'], driver['lng'])`
        - `infoWindow: InfoWindow(title: driver['driver_name'] ?? 'Driver ${driver['driver_id']}', snippet: '$minutesAgo min ago')`
        - `icon: BitmapDescriptor.defaultMarkerWithHue(minutesAgo > 2 ? BitmapDescriptor.hueOrange : BitmapDescriptor.hueGreen)` — green for recent, orange for getting stale
    - Update `_driverMarkers` with new markers (replace all)
    - Update `_driverCount` and `_loading = false`
    - Call `setState`
    - On first load, if drivers exist, animate camera to show all markers using `_mapController?.animateCamera(CameraUpdate.newLatLngBounds(bounds, 50))`

    Build method:
    - AppBar: title "Live Tracking", subtitle showing "$_driverCount active drivers"
    - If `_loading`: show `Center(child: CircularProgressIndicator())`
    - If `_error`: show error with retry button
    - If no drivers: show `Center(child: Column(children: [Icon(Icons.location_off), Text("No active drivers")]))`
    - Otherwise: Stack with GoogleMap filling the screen
      - `GoogleMap(initialCameraPosition: CameraPosition(target: LatLng(-33.9249, 18.4241), zoom: 10), markers: Set<Marker>.from(_driverMarkers.values), onMapCreated: (controller) => _mapController = controller)`
      - Default camera position is Cape Town (-33.9249, 18.4241) — South African context per user profile
    - Bottom overlay chip showing "Last updated: X seconds ago" with a refresh IconButton

    dispose:
    - `_pollTimer?.cancel()`
    - `_mapController?.dispose()`

    The screen should handle the case where GET /drivers returns 403 for schedulers when `scheduler_gps_visible` is false — GpsService.getDriverLocations() returns empty list on error, so show the "No active drivers" state which is acceptable.
