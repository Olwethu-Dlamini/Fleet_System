       ```json
       {
         "name": "fleet-scheduler-e2e",

           "test:report": "npx playwright show-report"
       ```

    2. Install Playwright:
       `cd e2e && npm install && npx playwright install chromium`

    3. Create `e2e/playwright.config.js`:
       - Use `defineConfig` from `@playwright/test`
       - testDir: `.` (e2e root)
       - timeout: 30000
       - retries: 1
       - Use `baseURL` from env `E2E_BASE_URL` or default to `http://localhost:3000`
       - Set `extraHTTPHeaders: { 'Accept': 'application/json' }`
       - Reporter: `[['list'], ['html', { outputFolder: './report' }]]`
       - NO browser `projects` needed since we use apiRequestContext only
       - Add `testMatch: '**/*.spec.js'`

    4. Create `e2e/fixtures/auth.setup.js`:
       - Export a helper function `loginAs(apiContext, role)` that:
         a) POSTs to `/api/auth/login` with test credentials
         b) Returns the auth token
       - Define test user credentials per role (these match the seeded test DB):
         - admin: { username: 'admin', password: 'admin123' }
         - scheduler: { username: 'scheduler', password: 'scheduler123' }
         - technician: { username: 'technician', password: 'tech123' }
       - Export credentials so tests can override them via env vars:
         `process.env.TEST_ADMIN_USER || 'admin'`
         `process.env.TEST_ADMIN_PASS || 'admin123'`

    NOTE: E2E tests require a RUNNING backend server with a SEEDED database. They are NOT mocked. Add a comment at the top of each spec file: "// REQUIRES: live server on E2E_BASE_URL (default localhost:3000) with seeded test data"
