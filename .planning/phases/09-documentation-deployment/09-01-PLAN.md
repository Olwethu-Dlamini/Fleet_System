Create docs/user-manuals/admin-guide.md — the complete admin user manual for FleetScheduler Pro. The admin role has full access to all features.

Structure the manual with these sections:

1. **Getting Started** — Login process, first-time setup, navigating the app (bottom nav with 7 tabs: Dashboard, Jobs, Vehicles, Users, Assignments, Reports, Settings)
2. **Dashboard** — Jobs Today card with bar chart, badge counts, weekend filter toggle, drivers-assigned vs clients view toggle
3. **User Management** — Creating users (admin/scheduler/technician roles), editing users, contact phone numbers, viewing user list, permission differences between roles
4. **Vehicle Management** — Adding vehicles (name, license plate, type, capacity), editing, activating/deactivating, viewing vehicle list
5. **Vehicle Maintenance** — Scheduling maintenance (date range), viewing maintenance history, how maintenance blocks vehicle assignment
6. **Job Management** — Creating jobs (customer name/phone/address, job type, scheduled date/time, priority, description), editing, cancelling, viewing job list, job statuses (pending, assigned, in_progress, completed, cancelled)
7. **Job Assignment** — Assigning driver + vehicle to job, driver load balancing (green glow on low-load drivers, suggested chip), technician multi-assignment, vehicle swap (hotswap), assignment history
8. **Reports** — Available report types and how to generate/view them
9. **Notifications** — Notification bell icon, notification center (read/unread), notification types (job assigned, job starting soon, job overdue, time extension requests), email notification toggle
10. **Time Extensions** — Viewing time extension requests from drivers, impact analysis display, rescheduling suggestions, approve/deny workflow
11. **GPS and Live Tracking** — Live tracking map screen showing driver positions, directions/ETA on job detail, admin toggle for scheduler GPS visibility, GPS consent system overview
12. **Settings** — Admin settings screen (scheduler GPS visibility toggle, other configurable options)

Each section should include: what the feature does, step-by-step usage instructions, and any important notes (e.g., "vehicles in maintenance cannot be assigned to jobs during those dates").

Write in clear, non-technical language. Use markdown headers, bullet points, and numbered steps. Target audience is a fleet manager who is not a developer.

  <action>
Create two user manuals, adapting content from the admin guide to each role's permissions.

**docs/user-manuals/scheduler-guide.md** — Scheduler role can do everything admin can EXCEPT add/remove vehicles and users. Structure:

1. **Getting Started** — Login, app navigation (same 7 tabs but Users tab is read-only, Vehicle add/delete hidden)
2. **Dashboard** — Same as admin (bar chart, badges, weekend filter, toggle views)
3. **Job Management** — Full CRUD on jobs (same as admin)
4. **Job Assignment** — Assigning drivers/vehicles, load balancing visuals, vehicle swap, technician management
5. **Vehicle Maintenance** — Scheduling maintenance, viewing history (if permission allows; note: cannot add/remove vehicles)
6. **Notifications** — Bell icon, notification center, email toggle
7. **Time Extensions** — Reviewing driver time extension requests, viewing impact analysis, approve/deny workflow
8. **GPS and Live Tracking** — Live tracking map (only visible if admin has enabled GPS visibility for schedulers), directions/ETA on jobs
9. **Reports** — Generating and viewing reports
10. **Key Differences from Admin** — Summary of what schedulers cannot do (add/remove users, add/remove vehicles, toggle scheduler GPS visibility)

**docs/user-manuals/driver-technician-guide.md** — Driver/technician role has limited access. Structure:

1. **Getting Started** — Login, GPS consent screen (shown on first login, required for POPIA/GDPR compliance), app navigation (limited tabs: Dashboard, My Jobs, Notifications)
2. **Viewing Assigned Jobs** — Job list showing only jobs assigned to you, job detail screen with customer info, map/directions, ETA
3. **Updating Job Status** — How job status changes (pending -> assigned -> in_progress -> completed), what each status means
4. **Completing a Job** — Confirm dialog, GPS capture on completion (coordinates recorded), what happens after completion
5. **Requesting Time Extensions** — "Add More Time" button (only visible on in_progress jobs you are assigned to), filling in the request form with reason and extra time needed, tracking request status
6. **Notifications** — Notification bell, types you receive (job assigned to you, schedule changes, time extension approved/denied), managing notification preferences
7. **GPS Tracking** — Consent screen explanation, what data is collected and when (only during active jobs/working hours), how to grant/decline consent, what happens if declined
8. **Troubleshooting** — Common issues (GPS not working, notifications not arriving, cannot see assigned jobs)

Write both in clear, non-technical language with numbered steps. Match the tone and formatting of the admin guide.
  </action>
