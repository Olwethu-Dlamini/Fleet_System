For each of the 14 route files, read the file to discover every route handler (router.get, router.post, router.put, router.patch, router.delete), then add JSDoc `@swagger` comment blocks above each handler.

Each @swagger block must include:
- `tags` — group by route file (e.g., "Authentication", "Jobs", "Vehicles")
- `summary` — one-line description
- `description` — detailed explanation of what the endpoint does
- `security` — `[{ApiKeyAuth: []}]` for protected routes (all except POST /auth/login and POST /auth/register)
- `parameters` — path params, query params with types and descriptions
- `requestBody` — for POST/PUT/PATCH with `application/json` content and schema
- `responses` — at minimum 200 (success with example response body), 400 (validation error), 401 (unauthorized), 404 (not found where applicable), 500 (server error)

Example annotation pattern to follow:
```javascript
/**
 * @swagger
 * /auth/login:
 *   post:
 *     tags: [Authentication]
 *     summary: Login with email and password
 *     description: Authenticates a user and returns a JWT token
 *     requestBody:
 *       required: true
 *       content:
 *         application/json:
 *           schema:
 *             type: object
 *             required: [email, password]
 *             properties:
 *               email:
 *                 type: string
 *                 example: admin@fleet.com
 *               password:
 *                 type: string
 *                 example: password123
 *     responses:
 *       200:
 *         description: Login successful
 *         content:
 *           application/json:
 *             schema:
 *               type: object
 *               properties:
 *                 token:
 *                   type: string
 *                 user:
 *                   type: object
 *       401:
 *         description: Invalid credentials
 */
```
